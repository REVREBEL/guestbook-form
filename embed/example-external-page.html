External Guestbook Embed Example

ℹ️ Note: This is an example of how to embed the Guestbook button on an external page (outside of the Webflow Cloud app).
⚠️ Important: Before this works, you need to:
  1. Build the embed/guestbook-embed.tsx file into a bundled JavaScript file
  2. Host the bundled file on a CDN or your server
  3. Replace BUNDLED_SCRIPT_URL below with the actual URL
  4. Replace YOUR_APP_BASE_URL with your deployed Webflow Cloud app URL

Embedded Guestbook Button:

How to Build the Embed Bundle

You'll need to bundle the embed/guestbook-embed.tsx file into a standalone JavaScript file.

Option 1: Using Vite

// vite.config.embed.js
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
  build: {
    lib: {
      entry: './embed/guestbook-embed.tsx',
      name: 'GuestbookEmbed',
      fileName: 'guestbook-embed',
      formats: ['iife']
    },
    rollupOptions: {
      external: [],
      output: {
        globals: {}
      }
    }
  }
});

// Then run:
// npx vite build --config vite.config.embed.js

Option 2: Using Webpack

// webpack.embed.config.js
module.exports = {
  entry: './embed/guestbook-embed.tsx',
  output: {
    filename: 'guestbook-embed.js',
    library: 'GuestbookEmbed',
    libraryTarget: 'umd'
  },
  module: {
    rules: [
      {
        test: /\.tsx?$/,
        use: 'ts-loader',
        exclude: /node_modules/
      },
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader', 'postcss-loader']
      }
    ]
  },
  resolve: {
    extensions: ['.tsx', '.ts', '.js']
  }
};

// Then run:
// npx webpack --config webpack.embed.config.js

Deploy the Bundle

After building, upload the bundled JavaScript file to:

  • A CDN (like Cloudflare, AWS S3 + CloudFront, etc.)
  • Your own server
  • Webflow hosting (if available for custom scripts)

Set the baseUrl

The baseUrl should point to your deployed Webflow Cloud app, for example:

  • https://your-site.webflow.io/guestbook-app
  • https://yourdomain.com/app

This is needed because the embed will make API calls to {baseUrl}/api/cms/...
